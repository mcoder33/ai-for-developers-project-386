# Интеграционные сценарии

## Основной сценарий бронирования

1. Гость открывает публичную страницу записи.
2. Гость видит доступные типы встреч и выбирает `Intro call`.
3. Приложение открывает страницу выбора времени и показывает свободные слоты.
4. Гость выбирает первый доступный слот, вводит имя и email.
5. Гость отправляет форму бронирования.
6. Приложение показывает экран успешной записи с типом встречи и данными гостя.
7. Владелец открывает список предстоящих встреч и видит созданное бронирование.

Проверка зафиксирована в `frontend/tests/e2e/booking.spec.ts`. Тест запускает реальный Go API и Vite-приложение через Playwright `webServer`, поэтому сценарий проходит через браузер, фронтенд и бэкенд.
